Project contextâ€‹

One of my greatest desires over the summer was to make a personal website to show off the LLM toy project that I made and was quite proud of, as well as to document more of my work to show to recruiters and friends. Although I've made personal websites in the past, using Google Sites and Wix.com, none of these websites really satisfied me or made me feel proud enough to put the work on a resume. Another systematic challenge with these website makers is that they have low customizability and it takes a lot of time to really dial in the style you want with their limited GUI based design systems. Another set of challenges is that these websites usually have is that they not future proof and what I specifically mean by that is that their content is not machine readable or easily accessible to LLMs that have been trained for web tools. I recognize this as something that will be critical in the future because I speculate recruiters may not even be looking at my profile or website. Instead, they might just be using an army of AI agents to do the work and assess whether or not I am a qualified candidate for whichever role I'm applying to. Another set of challenges that legacy website development has had is that it's very difficult to create a blog. And this is something I'm really interested in because I have a lot of ideas and I feel like I just need some place on the internet. Where a select group of people may get the access to see what I'm thinking about, whether it be a recruiter or some of my friends. Website builders with GUI based design systems are usually ugly or they just don't have this function of adding a blog. For a full on custom website using HTML, I just don't have that option because I'm not a computer science student and I don't have the time to learn all of the necessary front-end development software skills to make it possible for me to complete such a project while also studying to become a mechanical engineer. However, I do want to learn how to acquire some very basic and but very essential computer science skills that would make my skill set as a mechanical engineer significantly stronger and for me to be a much more valuable employee because of the intersectional skills that I know.

The Solution: Docusaurus When I found about Docusaurus over the summer, I became so excited. It was like the perfect thing. It is a static site generator that is built to make documentation sites, but it could very easily be used to createÂ any kind of siteÂ (personal website, product, blog, etc). But the best things about it from my perspective is that it's open source so all of my data is secure and within my control and it lets me focus on my work by using markdown files to host content. Also Markdown files are highly suitable for use with LLMs due to their simplicity and readability, this would be great for making the site accessible to AI agents making it vastly more future-proof.

My Workflow for adding content to the siteâ€‹

To start crafting my addition to my personal website, I start on Obsidian, but first I made sure to enable the right files and link settings by disabling wiki links and making the default location for new attachments in a subfolder under the current folder. That subfolder would be called attachments. This way the file structure is formatted already in a way that is very compatible with Docusaurus.

Then simply go to the parent folders location in the MacOS Finder and drag and drop the selected projects parents folder into the "docs" or "blog" subfolder for the Docusaurus website on my VSCode Explorer window

There are usually some errors that arise with the formatting of text and images, for this I will make a local test build of the that is server on port 3000. After that I can visually see how the website will run when it is deployed onto GitHub, and fix any bugs that arise.

Deploying the website is simple, I point the terminal to the repo that hosts the submodule "my-website" then set my GitHub username and personal access token, then finally deploy using

npm run deploy

for the full details please reference the Docusaurus deployment guide at https://docusaurus.io/docs/deployment#deploying-to-github-pages

Coming full circle to the idea of an AI accessible personal project website, I wanted to make sure that my website was in fact accessible to AI. Not just accessible to AI, but very easily accessible to AI. So I used an offline LLM solution using LM Studio where I imported a small 4 billion parameter model that was trained in tool calling and gave it this prompt:

Prompt

Read to me what is on this site: https://ajaypal260.github.io/my-website/blog/first-blog-posthttps://ajaypal260.github.io/my-website/blog/first-blog-post

To give the model tool call abilities, I integrated a MCP server into LM Studio called web-search-mcp. Github: https://github.com/mrkrsl/web-search-mcp For safety, I made sure to turn on the settings where the model must always ask for my permission to go ahead with a web search.
