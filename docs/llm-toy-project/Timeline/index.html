Timeline

Development Timeline and Progress:​

NOTE

From April 3rd to April 12th, I used an Arduino MKR 1010 from a previous class project — the Floating Solar Tracker. During this time, I made only minor progress, primarily testing the functionality of various components such as the display, microphone, button, and other modules I had purchased online

Link to the project page: Floating Solar Tracker

Saturday April 12 at 3:35 PM:​

My ESP32 finally arrived! This marked a pivotal milestone in the project, as I could now begin serious firmware development.

Monday April 21 at 5:52 PM:​

Today we hit a major breakthrough — both the microphone and speaker are now working simultaneously on the same board, using the same firmware and button.

The only adjustment I had to make was simplifying the setup by removing the display. The system now reads microphone amplitude values correctly:

  • It responds to my voice, even at high volume.

  • I can plot these amplitude values in real time.

Next, I demonstrated the speaker functionality. By pressing a button, the device switches into speaker mode. This confirms the ESP32 can interface with both the mic and speaker simultaneously.

The only remaining issue is an unreliable button — it often fails to trigger the microphone correctly.

Monday April 21 at 8:11 PM:​

With high confidence that both the microphone and speaker were functioning and the firmware was stable, I moved to the next development phase: system integration.

I began by integrating the server software and firmware on the Arduino and drafted a high-level diagram showing how all components would interface.

Thursday April 24 at 9:31 PM:​

With AI-assisted drafting, I created a working proxy server prototype that connected to my LM Studio server where the LLM model was hosted — purely for testing purposes.

Friday April 25 at 4:20 PM:​

I successfully established a connection between the LLM Studio server and the proxy server. Additionally, I implemented a new UI for the program.

Sunday April 27, 2025 at 8:36 PM​

I switched to using Docker for the text-to-speech (TTS) and speech-to-text (STT) services on my proxy server. Initially, I believed this would simplify integration. However, I later abandoned this approach because Docker consumed too many system resources, significantly reducing performance and token throughput.

Tuesday April 29 at 7:11 PM​

I successfully compiled all the language models — TTS, STT, and LLM — natively on my MacBook M1.

Tuesday April 29 at 8:23 PM​

To improve performance, I investigated how the models were using my system’s hardware. I discovered the TTS model wasn’t using the GPU, so I began recompiling it for GPU acceleration.

This was a challenging process due to discrepancies between the documented and actual architectures. Debugging was crucial, and I relied heavily on macOS Activity Monitor to verify GPU usage.

Tuesday April 29 at 11:34 PM​

After individually compiling and testing each model, the next step was to integrate them into a single pipeline (STT → LLM → TTS). This was implemented within the proxy server to ensure a smooth user experience.

Thursday, May 1st 2025 at 6:39 PM​

The server successfully received microphone recordings from the device. I verified that the resulting WAV file could be transcribed by an LLM using the Aiko app, and it worked on the first attempt! This milestone significantly boosted my confidence in the project — the finish line was finally in sight. However, there was still a bug preventing the speaker from functioning with the current firmware, which I aimed to fix by the end of the night.

I then pivoted to designing a 3D-printed case for the device.

Friday, May 2nd, 2025 at 10:36 PM​

After searching online for an open-source speaker enclosure compatible with my EK1725 speakers and desired dimensions, I modified the design in Fusion 360 to include:

  • A microphone hole/tunnel
  • Modified side panels
  • A button mount
  • Removal of one speaker grill

I retained the original two-piece design (lid + body), which simplifies assembly and troubleshooting.

Hardware Enclosure

The enclosure for the Raspberry Pi in the LLM Toy System is based on a modified design by Pierre (Pierre_200611 on Printables). See the full attribution in the Acknowledgments & Credits section of this document for details on changes and licensing.

Saturday, May 3rd, 2025 at 2:29 PM​

The final assembled prototype was ready. With guests at my house, I had a perfect opportunity to showcase it. Unfortunately, the firmware and software were not yet functional, so the demo was purely a visual display.

Sunday May 4, 2025 at 10:36 PM​

The product was now fully functional — the LLM pipeline and proxy server worked correctly. The final step was to integrate the GUI into the main application.

Tuesday May 6th, 2025 at 8:07 PM​

The GUI was successfully integrated into the main software pipeline. At this stage, the entire project was fully operational, and even my sister was able to use it — a huge success!
